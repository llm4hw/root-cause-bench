The bug is that you declared a variable (variable1) outside any process or subprogram. In VHDL, variables are only allowed inside a process, function, or procedure (or must be declared as a shared variable in a package/architecture). The error message complains that a variable declared outside a subprogram must be marked as shared.

Why it's occurring:
â€¢ In VHDL, variables are meant to be used for sequential assignments and are local to a process or subprogram.
â€¢ Outside these regions (at the architecture level), you must either declare a signal for concurrent behavior or, if you really want a variable, declare it as a "shared variable" (which is rarely what you want in synthesizable code).
â€¢ The tool (test bench simulator) finds the variable declaration in the architecture section and complains because it is not inside a process.

How to fix:
â€¢ Move the variable declaration inside the process that uses it. For example:

  ------------------------------------------------------------------
  architecture Behavioral of top6 is
      signal signal1 : STD_LOGIC := '0';
  begin
      process (clk, rst)
          variable variable1 : STD_LOGIC := '0';  -- now inside the process
      begin
          if rst = '1' then
              signal1 <= '0';
              variable1 := '0'; 
          elsif rising_edge(clk) then
              variable1 := signal1;
          end if;
      end process;
  
      out1 <= signal1;
  end Behavioral;
  ------------------------------------------------------------------

â€¢ Alternatively, if you intend to have concurrent behavior, convert variable1 to a signal if that fits your design needs.

By moving the variable declaration inside the process (or converting it to a signal), you eliminate the error and conform to VHDL rules.
